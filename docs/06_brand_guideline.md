## 1. ブランドの本質

### 1.1 プロダクト定義（1文）
> **猫の一生に寄り添う、飼い主とシッターのマッチング＋コミュニティアプリ。**

### 1.2 存在意義（ミッション）
> **"猫との暮らしを、もっと豊かに、もっと安心に。"**
>
> すべての猫が、幸せに生きられる仕組みをつくる。

### 1.3 ビジョン（3〜5年後に目指す姿）
> **"猫を迎えた日から看取りまで、Nekomewsが隣にある。"**
>
> 猫に関する生活のすべてがここで完結する、猫の生活インフラになる。

### 1.4 バリュー（5つの約束）
| # | バリュー | 行動原則 |
|---|---|---|
| 1 | **猫ファースト** | すべての判断を「猫にとって良いか」で決める |
| 2 | **信頼を最優先** | スピードより安全・透明性・誠実さ |
| 3 | **温かみをデザインする** | 機能だけでなく感情も設計する |
| 4 | **コミュニティで育てる** | 運営が主役ではなく、ユーザーと保護団体が主役 |
| 5 | **技術で猫を救う** | AIとデータで一匹でも多くの猫の人生を良くする |

## 2. ブランド・パーソナリティ

### 2.1 人に例えるなら
> **「猫に詳しい優しいご近所さん」**
>
> 獣医じゃないけど頼れる。ペットショップの店員ほどビジネスライクじゃない。
> 友達ほどラフじゃない。**絶妙な距離感で寄り添ってくれる人。**

### 2.2 ブランドパーソナリティ軸

| 軸 | 度合い | 理由 |
|---|---|---|
| フォーマル ⇄ カジュアル | **7** カジュアル寄り | 飼い主の心理的安全性 |
| プロフェッショナル ⇄ フレンドリー | **6** フレンドリー寄り | ただし命を預かる責任感は保つ |
| モダン ⇄ トラディショナル | **5** 中庸 | 和のテイストも残す |
| 静か ⇄ 賑やか | **4** 静か寄り | 猫の性質に合わせた落ち着き |
| 真面目 ⇄ 遊び心 | **6** 遊び心あり | 肉球ボタンなど随所に |

### 2.3 「らしい」vs「らしくない」

| ✅ Nekomews らしい | ❌ らしくない |
|---|---|
| 「お帰りなさい、モモちゃんは元気に過ごしています」 | 「【重要】シッターからの業務完了通知です」 |
| 肉球型ボタン | 角ばった業務的UI |
| 保護猫への寄付導線 | 押し売り的オファー |
| 獣医監修の豆知識 | 根拠のないライフハック |
| エラー時「今はつながらないみたい。少し待ってね🐾」 | 「Error 500: Internal Server Error」 |
| シッターを「パートナー」と呼ぶ | シッターを「ワーカー」と呼ぶ |

## 3. ボイス & トーン

### 3.1 基本ボイス（不変）
- **一人称**：「私たち Nekomews」（法人色を出しすぎない・ユーザー視点で）
- **二人称**：「あなた」「飼い主さん」「シッターさん」
- **猫の呼び方**：「〇〇ちゃん／〇〇くん」（個体名重視）
- **敬語ベース**：ですます調を基本とし、親しみを混ぜる
- **絵文字**：🐾 と 🐈 は多用OK、ハート系・キラキラ系は抑制

### 3.2 トーン使い分け（シーン別）

| シーン | トーン | 例 |
|---|---|---|
| オンボーディング | わくわく・歓迎 | 「こんにちは🐾 猫との暮らし、もっと楽しくしていきましょう」 |
| 予約成立 | 安堵・温かい | 「予約が成立しました。〇〇さんが〇〇ちゃんをお世話します」 |
| トラブル通知 | 冷静・誠実 | 「ご心配をおかけしています。状況を確認中です」 |
| 支払い | 淡々・明瞭 | 「ご利用ありがとうございます。領収書はこちらから」 |
| レビュー依頼 | お願い調 | 「今回のシッティングはいかがでしたか？ 一言残してあげてください」 |
| キャンペーン告知 | 明るい・遊び心 | 「うちの子コンテスト、エントリー受付中🐾」 |
| 個人情報・規約 | フォーマル | 「個人情報の取り扱いについて、以下のとおりご説明いたします」 |

### 3.3 禁止表現

#### 🚫 絶対NG（法令違反リスク）
- 「最高級」「No.1」「唯一」「絶対安全」→ 景表法違反リスク
- 「治る」「効く」「症状が改善」→ 薬機法違反リスク（動物医薬品含む）
- 「プロのシッター」→ 資格未取得者には使わない
- 特定の競合社名を挙げて「〇〇より優れている」→ 比較広告NG
- 「誰でも簡単に稼げる」→ 副業系の景表法違反典型

#### 🚫 ブランドとしてNG
- 押し売り的オファー（「今だけ」「残り3名」など煽り強すぎるもの）
- 擬人化しすぎた猫のセリフ（「〇〇にゃん大満足！」など過度）
- 上から目線（「〜しましょう」の連発は避ける）
- 動物を「物」として扱う表現（「取引」「商品」など）
- ネガティブな動物関連表現（「捨て猫」→「保護を待つ猫」等に言い換え）

#### ⚠️ 注意
- 「シッティング」は業界用語だが一般には浸透していない →「猫のお世話」と併記
- 「マッチング」は恋愛アプリを連想する人もいる →「出会い」「ご紹介」を併用

#### カラー原則
- **飼い主と シッターの色は混ぜない**：モード切替時にクリアに変わる
- **赤は緊急時のみ**：通常のネガティブ（猫のストレス等）は使わない
- **パステル寄り**：原色は避ける（猫と親和性が低い）

### 4.2 タイポグラフィ

#### 日本語
- **見出し**：`Noto Sans JP Bold`
- **本文**：`Noto Sans JP Regular`

#### 英数
- **見出し・ロゴ**：`Inter Bold` or `Poppins SemiBold`
- **本文**：`Inter Regular`

#### スケール（既に詳細設計書に定義済）
- H1: 28 / H2: 22 / H3: 18 / Body: 15 / Caption: 13 / Small: 11

### 4.3 ロゴ方向性（詳細は `07_logo_direction.md` に別途）
- **コンセプト**：「猫」「つながり」「優しさ」を内包
- ワードマーク（Nekomews の文字組み）＋シンボル（肉球／猫のシルエット）
- カラーはアンバー基調のグラデーションを検討

### 4.4 写真・イラスト方針
- **写真**：自然光・家庭感・猫目線の低アングル推奨
- **避ける**：ケージ感・病院感・寒色系ライト・ステレオ笑顔の人
- **イラスト**：線が柔らかい手描き風／幾何学的すぎないもの
- **禁止**：AIで生成したとバレバレの「超リアルCG猫」

### 4.5 アイコン方針
- **outline と filled の2セット**
- 角丸：2pxで統一
- 線の太さ：1.5px
- Phosphor Icons / Lucide Icons を基本に、肉球アイコンのみカスタム

### 4.6 アニメーション
- **スプリング（ばね）系**：肉球ボタン、カードタップ
- **フェード**：画面遷移は慎重に（乗り物酔い対策）
- **所要時間**：200〜350ms 推奨、500ms 超えは避ける
- **動物虐待を連想する動き（捕まえる・追い回す）は禁止**

## 5. 書き方スタイル

### 5.1 文字・記号
- **句読点は全角の「、」「。」**
- **半角英数の前後にはスペースを入れない**（「iPhoneで撮影」）
- **数字は基本半角**（「3匹」、ただし口語で「サンビキ」と読ませたい場合のみ漢字）
- **スペースは半角1つ**（全角スペースは使わない）
- **カギ括弧は「」**（“”は英文内のみ）
- **猫の鳴き声**：「ニャー」「にゃー」の表記ゆれを避け「にゃあ」を基本

### 5.2 文字数ルール
| 要素 | 上限 |
|---|---|
| アプリ内CTAボタン | 8文字 |
| プッシュ通知タイトル | 30文字 |
| プッシュ通知本文 | 100文字 |
| LPヘッドライン | 25文字 |
| SNS投稿（X） | 140文字 |
| SNS投稿（Instagramキャプション） | 300文字 |
| メール件名 | 25文字 |

### 5.3 猫の名前の扱い
- **呼び捨てしない**：「モモ」ではなく「モモちゃん」
- **敬称は付与**：「モモちゃんがお家で待っています」
- **所有格**：「飼い主さんの猫」ではなく「〇〇さんのところの子」が温かい

## 6. シッター側のコミュニケーション

### 6.1 呼称
- **「シッターさん」「パートナーさん」**
- NG：「業者」「ワーカー」「配達員」

### 6.2 評価の扱い
- レビュー低評価時の連絡は **攻撃的でなく改善提案に**
- 例：「今回のご利用者様から〇〇についてのフィードバックがありました。次回改善点としてご確認いただければと思います」

### 6.3 収入まわり
- 「報酬」「稼ぐ」を連呼しない（副業詐欺アプリっぽくなる）
- 「活動のお礼として」「サービス料として」の言い方

## 7. ネガティブ・センシティブ時の対応

### 7.1 事故・疾病・死亡
- **絵文字NG・軽い表現NG**
- 「お気持ちお察しします」ではなく「このたびはご心痛、私たちも言葉が見つかりません」
- **相手の時間軸を尊重する**：こちらから頻繁に連絡しない

### 7.2 保護猫・センシティブなテーマ
- 「虐待」「殺処分」の言葉は必要な文脈でのみ使用
- 保護活動紹介は「一匹でも多くの命を繋ぐ」視点で
- 募金・寄付は「善行」を押し付けない

### 7.3 個人情報・プライバシー
- 運営側の軽口・裏話は原則禁止
- 社内のハンドリングミスも**誠実な謝罪** → **再発防止** の順

## 9. コンテンツ戦略の基本

### 9.1 コンテンツの3本柱
| 柱 | 比率 | 内容 |
|---|---|---|
| **1. 役に立つ** | 40% | 猫の健康・行動・フード・しつけ情報 |
| **2. エモーショナル** | 40% | うちの子自慢、保護猫の救出、心温まる話 |
| **3. サービス告知** | 20% | 新機能、キャンペーン、シッター紹介 |

### 9.2 投稿リズム目安
- X：1日1〜2投稿
- Instagram：週3〜4投稿
- TikTok：週2投稿
- note：月2本記事

### 9.3 禁止コンテンツ
- 政治・宗教・ジェンダー論争への介入
- 他ペット業界の炎上への便乗
- アルゴリズム目当てのゴシップ・陰謀論
- 有名人の無断使用（ミームも注意）
- AI生成と分からない形での使用

## 10. パートナー・協業の姿勢

### 10.1 提携先の優先順位
1. 保護団体（対等・支援姿勢）
2. 動物病院（プロフェッショナル尊重）
3. ペット保険会社（ビジネスパートナー）
4. ペットフードブランド（選別して紹介）
5. 広告主（最後・厳選）

### 10.2 保護団体への姿勢
- **こちらが支援する側**という立場を基本に
- 団体名を前面に出す（Nekomewsを目立たせない）
- 手数料は取らない／寄付モデルで成立

### 10.3 インフルエンサー連携
- **猫愛が本物の人を優先**（フォロワー数より関係性）
- ステマ規制厳守（#PR 明示必須）
- ギャランティは透明に

## 11. このガイドラインの運用
