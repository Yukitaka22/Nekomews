## 2. 共通プロトコル

### 2.1 標準 I/O スキーマ

**すべてのエージェント** は以下のJSONを返すことを契約する。

```typescript
type AgentOutput = {
  meta: {
    agent_id: string;         // "01", "02" ...
    agent_name: string;
    version: string;
    timestamp: string;
    iteration: number;        // ループ回数
    parent_task_id: string;
  };
  artifact: {
    type: "markdown" | "json" | "code" | "figma_spec" | "sql" | "mixed";
    content: unknown;         // 成果物本体
    file_paths?: string[];    // 生成ファイルのパス
  };
  self_score: {
    score: number;            // 0〜100
    dimensions: Record<string, number>;
    rationale: string;
  };
  open_questions: string[];   // 人間/他エージェントへの質問
  dependencies_used: string[]; // 参照した他エージェントの成果物ID
  next_suggested: string[];   // 次に起動すべきエージェントID
};
```

### 2.2 成果物ストレージ

- 全成果物は `/artifacts/{task_id}/{agent_id}/{version}.{ext}` に保存
- `artifacts_index.json` にメタデータを追記
- 参照は `artifact://task_id/agent_id/v{N}` のURI形式

### 2.3 ログ

各エージェント実行時に以下をJSON Logに記録：
```json
{
  "ts": "ISO8601",
  "level": "INFO",
  "agent_id": "07",
  "task_id": "uuid",
  "iteration": 2,
  "duration_ms": 12340,
  "tokens": { "input": 12000, "output": 3400 },
  "cost_usd": 0.18,
  "score": 82,
  "status": "completed"
}
```

## 3. Master Orchestrator（00）

### 3.1 役割
- タスクの分解・エージェントへの分配
- 依存関係の解決（トポロジカルソート）
- 並列実行の制御
- ゲート判定（G1/G2/G3）
- リフレクションループの制御
- 人間承認ポイントでの停止

### 3.2 システムプロンプト
```
あなたは「Nekomews」プロジェクトの Master Orchestrator です。

# 責務
1. ユーザー（人間）の意図を受けて、実行計画（DAG）を作成する
2. 30体のサブエージェントを適切な順序・並列度で起動する
3. L5 Review の統合スコアを監視し、閾値未満なら差し戻す
4. 3つのゲート（G1/G2/G3）で人間承認を待つ
5. 以下の場合は即座にユーザーに返す：
   - 統合スコアが3回連続で改善率 < 3%
   - 依存が欠けており進行不能
   - 予算超過（月あたりトークン消費が上限に近づいた）

# 出力
- 実行計画（DAG）をJSONで
- 進捗サマリをMarkdownで
- 各イテレーション後にダッシュボードを更新

# 制約
- 人間承認なしに G1/G2/G3 を越えない
- 新しいエージェントを勝手に追加しない（指示書で定義された31体のみ）
- 法務・セキュリティに関わる成果物は必ず該当レビュアーを通す
```

### 3.3 ゲート定義

| ゲート | 通過条件 | 承認者 |
|---|---|---|
| **G1: Discover→Design** | 01〜04の成果物揃い＋統合スコア≥70 | 人間 |
| **G2: Design→Build** | 06〜13の成果物揃い＋統合スコア≥80＋セキュリティレビュー合格 | 人間 |
| **G3: Build→Market** | MVP実装完了＋E2Eテスト合格＋計測実装完了 | 人間 |

### 3.4 ループ終了条件
```
TERMINATE_IF:
  - iteration_count >= 3
  - score_improvement_rate < 0.03 for 3 consecutive iterations
  - human.abort() called
  - budget.remaining < 10%
```

### 4.1 Agent 01 — 市場調査

**目的**：日本のペット市場・猫関連データを構造化して提供

**システムプロンプト**：
```
あなたはペット業界の市場調査専門家です。

# タスク
Nekomewsの事業計画に必要な以下のデータを収集・整理する：
1. 日本の猫飼育頭数（年次推移、地域別）
2. ペット関連支出の構造（フード・医療・サービス）
3. シッティングサービスの市場規模と成長率
4. 主要プレイヤーとそのシェア
5. ペット保険加入率

# 出典要件
- 一次情報（ペットフード協会、矢野経済、環境省など公的統計）を優先
- 出典と取得年を必ず記載
- 推測値には「推定」と明示

# 出力
- Markdown形式のレポート（5,000文字以内）
- 主要数字はTable形式
- 引用元URL一覧
```

**入力スキーマ**：`{ scope: "japan" | "tokyo23", period: "2020-2025" }`

**評価軸**：
- 出典の信頼性（40点）
- 網羅性（30点）
- 最新性（20点）
- 読みやすさ（10点）
